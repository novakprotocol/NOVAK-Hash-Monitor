APPENDIX F — HIGH SCHOOL LEVEL EXPLANATION OF THE NOVAK PROTOCOL
A simple, clear explanation of NOVAK for non-technical readers
F.1 What Is NOVAK? (Simple Version)

NOVAK is a system that forces computers, robots, AI, and government programs to prove they are doing the right thing BEFORE they act.

Today, most systems do this:

Act → Then Check


NOVAK makes them do this:

Check → Then Act


That small change prevents:

mistakes

fraud

wrong decisions

bad data

unfair treatment

broken rules

glitches

hacking

Because nothing is allowed to happen unless it has already been proven correct.

F.2 Why Do We Need Something Like NOVAK?

Because computers and automated systems sometimes:

get things wrong

use incorrect data

ignore important rules

behave differently each time

crash or miscalculate

get tricked by people

get hacked

NOVAK makes sure these problems are caught before an action happens.

F.3 How Does NOVAK Work? (Simple Version)

Every time a system wants to take an action (like approving a benefit, controlling a robot, deciding an AI result, etc.), NOVAK checks:

Is the rule correct?

Is the data correct?

Is the output predictable and correct?

Is the person or machine really who they say they are?

Is the physical machine stable?

Is the human user being honest?

Is the decision happening at the right time?

Is this consistent with everything that happened before?

If all these checks pass → the action is allowed.
If any one fails → the action is blocked.

NOVAK has 15 Laws that every system must follow. Here they are in simple English:

You can’t change anything after the fact.

Output must be predictable.

Every action connects to the one before it.

Anyone can verify what happened.

The system shouldn’t rely on trust.

Together, these Laws make sure everything is:

fair

consistent

accurate

safe

trustworthy

F.5 What Are the Main Parts of NOVAK?

NOVAK has four main parts. Here’s what each one does (in simple terms):

1. Safety Gate

Checks EVERYTHING before an action happens.
If something is wrong → stops the action.

2. Execution Identity Receipt (EIR)

A digital “receipt” proving:

the rule

the input

the identity

the predicted output

were all correct.

3. HVET

A fancy hash (like a digital fingerprint) that ties all the data together so it can’t be faked.

4. RGAC

A chain that links every action in order, so nothing can be hidden or changed later.
